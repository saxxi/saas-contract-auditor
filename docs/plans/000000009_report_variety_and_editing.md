# Plan: Report Visual Variety + Inline Section Editing

**Status: Implemented** (2026-03-08)

## Context

Reports generated by plan 000000008 look polished but identical across all proposition types. Only the banner color varies — the section order, chart types, accent colors, and component styling are uniform. Additionally, editing was reduced to raw markdown mode (MDEditor) because the visual preview is read-only. Users lost the ability to intuitively edit report content.

**Goals:**
1. Make each proposition type visually distinct (different charts, colors, section emphasis, action callouts)
2. Restore editing with inline section-level editing in the visual preview

---

## Phase 1: Theme System + Visual Differentiation

### New file: `report-preview/theme-config.ts`

Centralized theme config keyed by `PropositionType`. Each theme defines:
- **Colors**: primary, accent, muted Tailwind classes (threaded to all sub-components)
- **Section order**: which sections render first (e.g., "requires negotiation" leads with Risks; "upsell" leads with Resolution)
- **Hero chart type**: which chart component appears prominently in Key Metrics
- **Action callout config**: label, color, what data to highlight

| Proposition Type | Primary | Hero Chart | Lead Section | Action Callout |
|---|---|---|---|---|
| requires negotiation | red-600 | Horizontal bar (usage vs limit with red overflow) | Risks | "Immediate Intervention Required" + breached limits |
| upsell proposition | blue-600 | Grouped bar (current vs next-tier headroom) | Resolution | "Revenue Opportunity" + estimated uplift |
| poor usage | amber-500 | Area sparkline (declining trajectory) | Metrics | "Engagement At Risk" + lowest utilization |
| at capacity | orange-500 | Multi-ring radial (3 metrics in concentric rings) | Metrics | "Capacity Threshold Approaching" + highest % |
| healthy | emerald-600 | Simple green stat cards (calm, minimal) | Exec Summary | No urgent callout; subtle "On Track" badge |

### New chart components (all recharts, already installed)

- **`bar-comparison-chart.tsx`** — `BarChart` for upsell/negotiation. Shows current vs limit with visual overflow
- **`capacity-radial.tsx`** — Multi-ring `RadialBarChart` for "at capacity". Three concentric rings for users/invoices/integrations
- **`usage-trend-spark.tsx`** — Small `AreaChart` sparkline for "poor usage" showing declining utilization

### New: `action-callout.tsx`

Themed callout banner between Executive Summary and next section. Different visually per proposition type — urgency level reflected in color, border style, and messaging.

### Modified sub-components (accept theme colors)

All existing components get a `themeColor` prop to replace hardcoded colors:
- `section-header.tsx` — accent line color from theme
- `option-card.tsx` — recommended card gradient from theme primary (not always amber)
- `risk-matrix.tsx` — row tint intensity varies (bolder for negotiation)
- `timeline-steps.tsx` — circle gradient from theme primary (not always blue)
- `evidence-card.tsx` — left border from theme accent (not always blue)
- `key-question.tsx` — left border from theme accent

### Modified: `report-preview.tsx`

- Look up theme by `propositionType`
- Render sections in theme-defined order (build render list from `sectionOrder` array)
- Insert hero chart in Key Metrics section based on theme
- Insert action callout between Executive Summary and next section
- Thread theme to all sub-components

## Phase 2: Inline Section Editing

### Modified: `parse-report.ts`

- Add `startLine` and `endLine` to `ParsedSection` (tracked during parsing)
- New helper: `replaceSectionBody(fullMarkdown, section, newBody): string` — splices edited section back into full markdown

### New: `editable-section.tsx`

Wrapper component:
- **View mode**: renders children, shows pencil icon on hover (top-right fade-in)
- **Edit mode**: replaces children with a compact `@uiw/react-md-editor` scoped to that section's markdown body
- Save/Cancel buttons. Escape key cancels
- Only one section editable at a time (state managed by parent)
- Blue highlight border around section being edited

### Modified: `report-preview.tsx`

- New props: `editable?: boolean`, `onContentChange?: (newMarkdown: string) => void`
- Wrap each section in `<EditableSection>` when editable
- On section save: call `replaceSectionBody()`, invoke `onContentChange` with updated full markdown

### Modified: `report-modal.tsx`

- Default mode: visual preview with inline editing enabled
- Pass `editable={true}` and `onContentChange={handleContentChange}` to `ReportPreview`
- Rename "Edit" button to "Raw Markdown" (escape hatch for full control)
- Remove binary `isEditing` state; use `mode: "preview" | "raw"` instead

## Phase 3: LLM Prompt Enhancement

### Modified: `apps/agent/src/prompts.py`

Add proposition-type-specific hints so LLM content itself varies (not just presentation):
- **upsell**: include estimated revenue uplift in Resolution
- **negotiation**: lead Complication with specific limit breaches and dollar impact
- **poor usage**: call out lowest utilization dimension, suggest engagement tactics
- **at capacity**: quantify remaining headroom before limits hit
